# CLAUDE.md

## Project Overview

Personal blog at [chrishannah.me](https://chrishannah.me), built with Eleventy (11ty) v3 as an ES module. Dark/light/auto theme via CSS custom properties. Outputs static HTML to `public/`, deployed on Vercel from `master` (auto-deploy on push).

## Build & Dev Commands

```bash
npm run dev                 # Dev server with live reload; skips old posts for speed
npm start                   # Dev server (full build, no dev optimizations)
npm run build               # Production build (eleventy && pagefind --site public)
./new_post.sh "Post Title"  # Create a new post (opens in Neovim)
./new_link.sh               # Create a new link post in inbox-links/
./process-inbox.sh          # Process inbox/ and inbox-links/ into posts/
```

**Search caveat:** `npm run build` also runs Pagefind to index the built site into `public/pagefind/`. The dev server (`npm run dev` / `npm start`) does **not** run Pagefind, so `/search/` only returns results after a full `npm run build`.

## Architecture

### Theme: Steno

Dark/light/auto, monospace-leaning, warm palette (accent `#c87830` in dark, `#b85500` in light). CSS variables on `:root[data-theme=...]`.

Theme switch:
- `theme.js` reads `localStorage('theme-preference')`, sets `data-theme` on `<html>`, highlights the active toggle button.
- Inline script in each layout's `<head>` prevents flash of wrong theme on load.
- Optional **dynamic colors** mode: if an Instagram palette is available and `localStorage('dynamic-colors') === 'on'`, the inline head script overrides CSS vars with colors pulled from the latest Instagram photo.

### Layouts (`_includes/layouts/`)

| Layout | Used for |
|--------|----------|
| `base.njk` | All content pages ‚Äî full `<head>`, site header/nav, footer |
| `home-steno.njk` | Homepage + paginated blog index (minimal head, no per-page OG tags) |
| `post.njk` | Standard posts ‚Äî extends `base`, wraps content in `h-entry`, includes post-footer + transmit form |
| `link.njk` | Link posts |
| `micro.njk` | Microblog posts |
| `quote.njk` | Quote posts |
| `music.njk` | Music posts (artwork, artist metadata) |
| `page.njk` | Static pages (about, colophon, uses, etc.) ‚Äî no transmit form, no post-footer |
| `archive.njk` | Archive listings |

**The homepage (`/`) is a paginated blog index**, not a dashboard ‚Äî generated by `pages/blog.njk` with `home-steno.njk` as its layout. Pagination permalinks come from the `getBlogPermalinkSuffixFromPageNum` filter (provided by `eleventy-plugin-blog-tools`).

### Partials (`_includes/partials/`)

| File | Purpose |
|------|---------|
| `post-footer.njk` | Author byline, permalink, source (link posts), reply-via-email. Marked `data-pagefind-ignore` so the boilerplate stays out of search. |
| `post.njk`, `link.njk`, `micro.njk`, `quote.njk` | Rendering blocks for each post type |
| `post-nav.njk` | Newer/Older (same post type) + up to 3 related posts (shared tags). Included by `post.njk` and `link.njk`; expects `navCollection` set before include. |
| `living-document-notice.njk` | Shown on pages with `living: true` frontmatter |
| `transmit.njk` | "Transmit" message form (ntfy-backed) ‚Äî included at bottom of `post.njk` |
| `steno/footer.njk` | Sitewide footer: pages column (incl. Tags, Search), projects column, contact column, "Now" column with Minifocus + Instagram |

### CSS (`assets/css/`, inlined via `{% include %}`)

| File | Purpose |
|------|---------|
| `steno.css` | Main stylesheet ‚Äî variables, layout, typography, forms, header/nav/footer, cards, post entries, pagination, archive, 404, transmit, heading anchors, tag cloud, Pagefind UI theming, post-nav/related |
| `fonts.css` | Currently empty ‚Äî reserved for future self-hosted `@font-face` declarations. |
| `highlight.css` | Code syntax highlighting. Inlined in `base.njk` **only when the page contains a `<pre>` block** (paired with highlight.js, injected on `<code>`). |

`base.njk` content-sniffs the rendered child (`'<pre' in content`, `'<img' in content`) to decide whether to inline `highlight.css` / `lightbox.css`, so pages that don't need them stay lean.

### Content Structure

Posts live in `posts/YYYY/MM/` as Markdown. Frontmatter:

```yaml
---
title: Post Title
date: YYYY-MM-DD
tags:
  - post                      # or: micro, link, essay, quote, photography, music, note
layout: layouts/post
permalink: slug-here/
draft: true                   # optional, excluded via excludeDrafts filter
categories: ["Category"]      # optional
image: /path/to/hero.jpg      # optional
link: https://...             # link posts: external URL
artist: "Artist Name"         # music posts
artwork: /path/to/cover.jpg   # music posts
---
```

Tag semantics:
- `post` ‚Äî standard blog post
- `micro` ‚Äî short microblog entry
- `link` ‚Äî link post (title links externally)
- `essay` ‚Äî long-form essay
- `quote` ‚Äî quote post
- `photography` ‚Äî photo post
- `music` ‚Äî music log entry
- `note` ‚Äî photo of a field-notes notebook page (title + `image:`, no body)

### Collections (`eleventy.config.js`)

- `all` ‚Äî all Markdown posts (excludes pages)
- `essay`, `link`, `micro`, `photography`, `music`, `note`, `quote` ‚Äî filtered by tag
- `tagList` ‚Äî normalised, browsable tags. One entry per slug (`{ slug, label, count, posts }`); casing variants (iOS/ios, "Text Case") merge, canonical label = most common casing. Excludes content-type tags. Drives `/tags/`.

Markdown: `markdown-it` + `markdown-it-footnote` + **`markdown-it-anchor`** (assigns heading ids via the shared `config/slugify.js`, with a hover-revealed `#` permalink). This is what makes the `toc` filter and heading deep-links resolve.

Custom filters added in config:
- `excludeDrafts` ‚Äî filter `draft: true`
- `limit` ‚Äî array slice
- `toc` ‚Äî table of contents; reads the real heading ids assigned by markdown-it-anchor
- `buildDuration` ‚Äî seconds since build start
- `filterTagList` ‚Äî strip content-type tags from display
- `tagSlug` ‚Äî normalise a tag to its `/tags/<slug>/` slug (same slugify as `tagList`)
- `relatedByTags` ‚Äî rank posts by shared normalised tags (per-post slug cache)
- Plus filters from `config/date.js`, `config/number.js`, `config/post.js`, `config/stats.js`

### Config Filters (`config/`)

| File | Filters |
|------|---------|
| `date.js` | Date formatting (uses `moment`) |
| `number.js` | Number formatting |
| `post.js` | `metaDescription`, `readingTime`, `stripHeaderAnchors`, `excerpt`, `microExcerpt`, `cleanUrl` |
| `slugify.js` | Shared heading/tag slug function (used by markdown-it-anchor, `toc`, `tagSlug`, `tagList`, `relatedByTags`) |
| `stats.js` | `daysPublishing`, `postsThisMonth`, `postsThisYear`, `daysSinceLastPost`, `volumeNumber`, `nextMilestone`, `longestStreak`, `bestMonth` |

`metaDescription` (used in both `<head>`s) extracts clean body text from `.content`/`.e-content`/`.post-body` via cheerio so page chrome doesn't leak into `<meta description>`/`og:description`. `stripHeaderAnchors` keeps the anchor `#` permalinks out of RSS content.

### Inbox Workflow

Drafts can land in `inbox/` or `inbox-links/` (ignored by Eleventy builds). `process-inbox.sh` moves them into `posts/YYYY/MM/` with correct frontmatter. A GitHub Action mirrors this server-side.

### Pages (`pages/`)

Static pages (`about.md`, `colophon.md`, `uses.md`, etc.). The `all` collection excludes anything with `page` in its layout, so these don't pollute post lists.

- `tags.njk` ‚Üí `/tags/` ‚Äî tag cloud (all of `collections.tagList`, by count).
- `tag.njk` ‚Üí `/tags/<slug>/` ‚Äî paginated one-per-tag, lists the tag's posts via `archive-list.njk`.
- `search.njk` ‚Üí `/search/` ‚Äî Pagefind UI. Only content articles carry `data-pagefind-body` (in the leaf layouts), so posts and static pages are indexed but listing pages (homepage, archives, the ~460 tag pages) are not.

### Build Ignores

`eleventy.config.js` ignores: top-level `*.md` / `*.MD`, `inbox/`, `inbox-links/`, `blog-editor/`, `api/`, `ghost-export/`.

### Dev Mode Speedup

When `ELEVENTY_ENV=development`, posts from years older than last year are ignored (dynamically computed) ‚Äî makes incremental dev server much faster.

## Project-Specific Learnings

### Site navigation lives in layouts, not a partial
- Content pages: `_includes/layouts/base.njk` (`<nav class="site-nav">`)
- Homepage: `_includes/layouts/home-steno.njk`

There's no shared header partial ‚Äî the `steno/` partials dir only holds `footer.njk`. If you're editing nav, edit both layouts.

### Homepage ‚âÝ dashboard
`/` is a paginated blog index (`pages/blog.njk` + `home-steno.njk`), not a status panel. Change content in `pages/blog.njk`, chrome in `home-steno.njk`.

### Two `<head>` sections
`base.njk` (content pages) and `home-steno.njk` (homepage) have separate `<head>` blocks with different meta tags, OG image logic, and font loading. Sitewide head changes must go in both.

### Don't duplicate between `projects.json` and `external.js`
Homepage and footer render them as separate sections. An item in both shows up twice.

### Tinylytics has no CORS
Can't be fetched from the browser. Use their embed script ‚Äî add `<span class="tinylytics_hits">` / `<button class="tinylytics_kudos">` placeholders and the embed fills them. `setInterval` poll if you need to read values from JS (embed loads async).

### Behold.so response shape
`data.posts[0]`, not a flat array. Prefer `sizes.medium.mediaUrl` (stable) over raw `mediaUrl` (signed URL that expires).

### Filter performance
Filters called from `base.njk` / `home-steno.njk` run on every rendered page ‚Äî hundreds of calls per build. Cache aggressively when inputs don't change (see `postsThisYear` in `config/stats.js` for the pattern).

### Tags are normalised by slug
493 raw tags with inconsistent casing collapse to ~460 `/tags/<slug>/` pages. `tagList`, the `tagSlug` filter, `relatedByTags`, and the pill links all run tags through `config/slugify.js`, so they always agree ‚Äî change the slug rule in **one** place. The canonical label shown is the most common original casing.

### Search indexing is opt-in per article
Pagefind only indexes elements carrying `data-pagefind-body`, which lives on the `<article>` in the *leaf* layouts (`post`/`link`/`micro`/`quote`/`music`/`page`). Listing pages (homepage, archives, tag pages, `/search/`) deliberately omit it so search returns content, not indexes. `post-footer.njk` is `data-pagefind-ignore`d so "Permalink / Reply via Email" doesn't pollute results.

### Heading anchors feed RSS too
`markdown-it-anchor` adds a `#` permalink to every rendered heading, including content that flows into feeds. The RSS templates pipe `templateContent` through `stripHeaderAnchors` to remove them. If you add a new feed, include that filter.

### Miniroll embed
Supports `data-color-mode="light|dark|auto"`. Set the attribute **before** the script loads ‚Äî create the `<script>` dynamically once theme is resolved. When toggling theme after load, update the iframe's `color_mode` query param.

### Workflow
- Feature branches over direct commits to `master`.
- Fizzy task descriptions are plain text (no Markdown rendering) ‚Äî use blank lines for paragraph breaks.
- Verify behavior on the Vercel preview URL before merging UI changes.
- Update CLAUDE.md after significant architectural changes so it stays accurate.
