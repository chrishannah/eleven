# CLAUDE.md

## Project Overview

Personal blog at [chrishannah.me](https://chrishannah.me), built with Eleventy (11ty) v3 as an ES module. Dark/light/auto theme via CSS custom properties. Outputs static HTML to `public/`, deployed on Vercel from `master` (auto-deploy on push).

## Build & Dev Commands

```bash
npm run dev                 # Dev server with live reload; skips old posts for speed
npm start                   # Dev server (full build, no dev optimizations)
npm run build               # Production build
./new_post.sh "Post Title"  # Create a new post (opens in Neovim)
./new_link.sh               # Create a new link post in inbox-links/
./process-inbox.sh          # Process inbox/ and inbox-links/ into posts/
```

## Architecture

### Theme: Steno

Dark/light/auto, monospace-leaning, warm palette (accent `#c87830` in dark, `#b85500` in light). CSS variables on `:root[data-theme=...]`.

Theme switch:
- `theme.js` reads `localStorage('theme-preference')`, sets `data-theme` on `<html>`, highlights the active toggle button.
- Inline script in each layout's `<head>` prevents flash of wrong theme on load.
- Optional **dynamic colors** mode: if an Instagram palette is available and `localStorage('dynamic-colors') === 'on'`, the inline head script overrides CSS vars with colors pulled from the latest Instagram photo.

### Layouts (`_includes/layouts/`)

| Layout | Used for |
|--------|----------|
| `base.njk` | All content pages — full `<head>`, site header/nav, footer |
| `home-steno.njk` | Homepage + paginated blog index (minimal head, no per-page OG tags) |
| `post.njk` | Standard posts — extends `base`, wraps content in `h-entry`, includes post-footer + transmit form |
| `link.njk` | Link posts |
| `micro.njk` | Microblog posts |
| `quote.njk` | Quote posts |
| `music.njk` | Music posts (artwork, artist metadata) |
| `page.njk` | Static pages (about, colophon, uses, etc.) — no transmit form, no post-footer |
| `archive.njk` | Archive listings |

**The homepage (`/`) is a paginated blog index**, not a dashboard — generated by `pages/blog.njk` with `home-steno.njk` as its layout. Pagination permalinks come from the `getBlogPermalinkSuffixFromPageNum` filter (provided by `eleventy-plugin-blog-tools`).

### Content Structure

Posts live in `posts/YYYY/MM/` as Markdown. Frontmatter:

```yaml
---
title: Post Title
date: YYYY-MM-DD
tags:
  - post                      # or: micro, link, essay, quote, photography, music
layout: layouts/post
permalink: slug-here/
draft: true                   # optional, excluded via excludeDrafts filter
categories: ["Category"]      # optional
image: /path/to/hero.jpg      # optional
link: https://...             # link posts: external URL
artist: "Artist Name"         # music posts
artwork: /path/to/cover.jpg   # music posts
---
```

Tag semantics:
- `post` — standard blog post
- `micro` — short microblog entry
- `link` — link post (title links externally)
- `essay` — long-form essay
- `quote` — quote post
- `photography` — photo post
- `music` — music log entry

### Collections (`eleventy.config.js`)

- `all` — all Markdown posts (excludes pages)
- `essay`, `link`, `micro`, `photography`, `music` — filtered by tag

Custom filters added in config:
- `excludeDrafts` — filter `draft: true`
- `limit` — array slice
- `toc` — generate table of contents from HTML headers
- `buildDuration` — seconds since build start
- `filterTagList` — strip content-type tags from display
- Plus filters from `config/date.js`, `config/number.js`, `config/post.js`, `config/stats.js`

| File | Purpose |
|------|---------|
| `site.json` | Site title, URL |
| `author.json` | Author info |
| `homepage.json` | Tagline, status, `since` year, current focus, Instagram username, music playlist ID |

**Projects vs external sites:** `projects.json` is software; `external.js` is other websites. The homepage and footer render them as separate sections — **don't duplicate** an item into both or it'll appear twice. The `/projects/` page pulls from both.

### Static Assets

`static/<folder>/` → `public/<folder>/` via passthrough copy. Folders: `text-shot`, `2020`, `cv`, `prntsc`, `fonts`, `images`, `videos`. Post images (`.jpg/.jpeg/.png/.gif`) inside `posts/**` are also passthrough-copied.

### Top-Level Templates

- `404.njk` → `/404.html`
- `robots.njk` → `/robots.txt`
- `sitemap.njk` → `/sitemap.xml`

### RSS Feeds (`rss/`)

- `/feed/index.xml` — all posts
- `/feed/posts/index.xml` — posts tagged `post`
- `/feed/essays/index.xml` — essays
- `/feed/micro/index.xml` — micro posts
- `/feed/music/index.xml` — music posts

### Inbox Workflow

Drafts can land in `inbox/` or `inbox-links/` (ignored by Eleventy builds). `process-inbox.sh` moves them into `posts/YYYY/MM/` with correct frontmatter. A GitHub Action mirrors this server-side.

### Pages (`pages/`)

Static pages (`about.md`, `colophon.md`, `uses.md`, etc.). The `all` collection excludes anything with `page` in its layout, so these don't pollute post lists.

### Build Ignores

`eleventy.config.js` ignores: top-level `*.md` / `*.MD`, `inbox/`, `inbox-links/`, `blog-editor/`, `api/`, `ghost-export/`.

### Dev Mode Speedup

When `ELEVENTY_ENV=development`, posts from years older than last year are ignored (dynamically computed) — makes incremental dev server much faster.

## Serverless API (`api/`)

Vercel serverless functions (ESM):

| File | Purpose |
|------|---------|
| `micropub.mjs` | IndieAuth Micropub endpoint. POST creates posts (commits Markdown to GitHub via `createFileInGitHub.mjs`); GET returns config. Handles regular posts and `like-of` favourites (with webmention sending). |
| `og.mjs` | OG image generator — `satori` + `resvg` rendering PNG cards with JetBrains Mono, dark background, orange accent. |
| `transmit.mjs` | Public "Transmit" message endpoint. POST from the `transmit.njk` form forwards `message` + optional `from` to ntfy as a push notification. Honeypot field, 500-char cap, control-char stripping, 303-redirect to `/transmitted/` on success. **Requires `NTFY_TOPIC` env var in Vercel** (optional `NTFY_SERVER`, defaults to `https://ntfy.sh`). |
| `buildPostContent.mjs` | Builds frontmatter + body for micropub-created posts |
| `createFileInGitHub.mjs` | Commits a new file to the repo via Octokit |
| `slugify.mjs` | Slug generation |
| `validateToken.mjs` | IndieAuth token validation |

## Project-Specific Learnings

### Site navigation lives in layouts, not a partial
- Content pages: `_includes/layouts/base.njk` (`<nav class="site-nav">`)
- Homepage: `_includes/layouts/home-steno.njk`

There's no shared header partial — the `steno/` partials dir only holds `footer.njk`. If you're editing nav, edit both layouts.

### Homepage ≠ dashboard
`/` is a paginated blog index (`pages/blog.njk` + `home-steno.njk`), not a status panel. Change content in `pages/blog.njk`, chrome in `home-steno.njk`.

### Two `<head>` sections
`base.njk` (content pages) and `home-steno.njk` (homepage) have separate `<head>` blocks with different meta tags, OG image logic, and font loading. Sitewide head changes must go in both.

### Don't duplicate between `projects.json` and `external.js`
Homepage and footer render them as separate sections. An item in both shows up twice.

### Tinylytics has no CORS
Can't be fetched from the browser. Use their embed script — add `<span class="tinylytics_hits">` / `<button class="tinylytics_kudos">` placeholders and the embed fills them. `setInterval` poll if you need to read values from JS (embed loads async).

### Behold.so response shape
`data.posts[0]`, not a flat array. Prefer `sizes.medium.mediaUrl` (stable) over raw `mediaUrl` (signed URL that expires).

### Filter performance
Filters called from `base.njk` / `home-steno.njk` run on every rendered page — hundreds of calls per build. Cache aggressively when inputs don't change (see `postsThisYear` in `config/stats.js` for the pattern).

### Miniroll embed
Supports `data-color-mode="light|dark|auto"`. Set the attribute **before** the script loads — create the `<script>` dynamically once theme is resolved. When toggling theme after load, update the iframe's `color_mode` query param.

### Workflow
- Feature branches over direct commits to `master`.
- Fizzy task descriptions are plain text (no Markdown rendering) — use blank lines for paragraph breaks.
- Verify behavior on the Vercel preview URL before merging UI changes.
- Update CLAUDE.md after significant architectural changes so it stays accurate.
